# AI 伴侣 App UX 文档生成方案

## Summary
基于 `PRDV0.1.md` 产出一份中文、Markdown 格式、移动端优先的交付型 UX 文档，建议文件为 `D:\Projects\AI\AiCompanion\UXD_v0.1.md`。文档目标不是展示概念，而是把 MVP 的交互方案定义到“设计、前端、产品都可直接执行”的粒度，重点覆盖“记忆 + 主动关心 + 关系连续性”三条主线。

## Key Changes
- 明确文档结构，按以下章节输出：
  1. 文档概述：版本、依据 PRD、目标用户、设计目标、适用范围
  2. 体验原则：被认识感、连续关系感、温和不越界、记忆可解释、用户可控
  3. 信息架构：移动端主导航与页面层级
  4. 核心用户旅程：首次建立关系、日常聊天、隔天回访、长时间未访问后的回流、记忆管理、高风险内容处理
  5. 页面级交互设计：逐页定义目标、布局、组件、操作、反馈、空态、异常态、边界态
  6. 关键机制规则：记忆触发、话题延续、主动问候、删除记忆后的行为约束、通知节流
  7. 文案与语气规范：朋友式陪伴、避免恋爱化和依赖感强化
  8. 验收清单：逐条映射 PRD 的成功标准与验收场景

- 先把信息架构定死，避免实现阶段再做产品决策：
  - 主入口采用 4 个一级区域：`聊天`、`会话`、`记忆`、`我的`
  - `聊天` 为默认首页，承担关系建立、连续对话、重要话题追问
  - `会话` 承担历史线程回访与恢复上下文
  - `记忆` 承担查看、编辑、删除与记忆来源解释
  - `我的` 仅保留必要设置：通知、称呼、隐私与安全说明，不扩展商业化和个性化复杂配置

- 页面级交互需要明确到可实现粒度：
  - 新用户引导拆为 4 步：欢迎页、称呼确认、兴趣收集、近期在意的事
  - 聊天主界面定义 6 类状态：首次空态、已建立关系常态、等待回复、触发记忆追问、触发主动问候承接、异常失败
  - 会话列表定义排序规则、摘要策略、恢复入口、空列表提示
  - 记忆中心定义记忆卡片结构、来源说明、编辑/删除确认、删除后影响提示
  - 主动问候定义两个触发流：固定时段问候、长时间未访问轻提醒，并要求消息落到具体上下文而非模板化寒暄
  - 高风险内容定义克制式响应、弱化依赖措辞、必要时引导现实支持资源

- 文档中要新增一组“规范性接口”，作为后续实现统一参照：
  - 页面 ID 与页面名称
  - 核心组件清单：消息气泡、记忆提示条、关系进展提示、问候卡片、记忆卡片、删除确认弹层
  - 关键事件清单：首次完成建联、话题续聊、记忆新增、记忆编辑、记忆删除、主动问候点击回流
  - 状态命名规范：空态、加载态、失败态、禁用态、撤销态  
  这不是代码 API，但会作为产品/设计/研发共用的交付接口。

## Implementation Details
- UX 文档采用“页面 + 流程 + 规则”三层写法：
  - 流程层负责回答“用户为什么会来到这里、下一步去哪里”
  - 页面层负责回答“页面上有什么、点了会怎样、失败时怎样”
  - 规则层负责回答“AI 在什么条件下表现出记忆、追问、主动关心”

- 默认交互策略直接锁定如下：
  - 以移动端单列布局为主，不做桌面端独立页面方案，只补充响应式适配原则
  - 首次引导不超过 4 步，全部问题都要可跳过或稍后补充，避免过重注册感
  - 聊天页必须有“为什么会提到这件事”的轻解释入口，支撑记忆可解释性
  - 主动消息只允许两类触发，不扩展复杂系统事件触发
  - 记忆编辑和删除都要给用户明确控制感，删除后不再引用相关内容
  - 关系表达只允许“温和朋友型”，禁止恋爱化、占有欲、依赖强化表达

- 文档表现形式也一并锁定：
  - 全文中文
  - 使用 Markdown 标题结构
  - 在关键流程处加入简化流程图或文字线框说明
  - 不输出高保真视觉稿，不定义品牌视觉系统，不展开技术实现细节

## Test Plan
- 文档完成后应能直接支持以下验收：
  - 设计或前端仅依赖该文档，就能画出首版页面草图并实现主流程
  - PRD 中 6 个必做功能均有对应页面或交互规则承接
  - PRD 中 6 条验收场景均能在文档中找到明确入口、页面行为和结果
  - 对首次使用、隔天回访、长期未访问、删除记忆、高风险情绪内容，都有单独的状态或流程定义
  - 文档中不存在超出 PRD 范围的功能扩张，如多角色、语音、商业化、Live2D、恋爱向设定

- 交付检查项：
  - 是否定义了一级导航与页面层级
  - 是否定义了每个核心页面的空态/加载态/异常态
  - 是否定义了主动问候的触发条件、消息形态和回流路径
  - 是否定义了记忆卡片的展示、修改、删除及影响
  - 是否定义了语气边界与安全边界

## Assumptions
- 采用移动端优先，Web/桌面端仅做适配原则，不做独立主方案
- 采用交付型文档，面向产品、设计、前端协作，不以答辩展示为优先
- 文档产物为单一 Markdown 文件，不生成 `.docx`、原型稿或视觉稿
- 默认文件名使用 `UXD_v0.1.md`，与当前 `PRDV0.1.md` 形成配套版本关系
- 当前仓库无其他 IA、视觉规范或组件库约束，因此由该 UX 文档直接承担首版交互基线
